# Git-стратегия — ÆON Map MAX Bot (соло, MVP)

Цель: не терять рабочее состояние и держать **main** деплойабельным, без лишней церемонии.

---

## Коммит и push

**Правило:** коммит (и push), когда изменения **работают и их не хочется терять** — не «когда всё идеально готово», а когда **стабильно**.

Практические триггеры:

| Ситуация | Действие |
|----------|----------|
| Написал функцию — **тест зелёный** | коммит |
| Закрыл **один** acceptance criterion из issue | коммит |
| Перед **рефакторингом** уже рабочего кода | коммит (**точка возврата**) |
| **Конец рабочего дня** — даже если задача не закрыта | коммит с префиксом **`wip:`** в сообщении |

**Частота:** в активной разработке **5–15 коммитов в день** для соло-проекта — **норма**, не «слишком много».

Сообщения коммитов: осмысленные одной строкой; `wip:` допускается для незавершённого, но сохранённого состояния.

---

## Ветки (минимализм для соло на MVP)

Отдельная ветка **не на каждый коммит**. Достаточно **двух** поводов:

1. **Фича дольше одного дня** и нужен стабильный `main` — ветка вида **`feat/<короткое-имя>`** (например `feat/rewards-engine`), **merge в main**, когда логически готово (например **итерация закрыта**).
2. **Эксперимент**, который могут выбросить — ветка **`exp/<тема>`** (например `exp/llm-classifier`).

**Всё остальное** — можно **прямо в `main`**. Feature-branch на каждый мелкий шаг на MVP с одним разработчиком — обычно лишняя церемония.

---

## Схема для этого репозитория

| Ветка | Назначение |
|-------|------------|
| **`main`** | Всегда **рабочее** состояние, **задеплоено** (или готово к деплою). |
| **`feat/iter-N`** | Работа по **итерации N** из roadmap (например `feat/iter-1` — Event Core, `feat/iter-2` — Dialog Engine). |

**Цикл:** закрыл итерацию → **merge в `main`** → **удалил ветку** → открываешь следующую `feat/iter-(N+1)`.

Так и человек, и **Cursor** явно видят контекст: «сейчас идёт итерация N в ветке `feat/iter-N`».

---

## Связь с Issues

PR с **`Closes #N`** по-прежнему; частые коммиты в ветке итерации дают историю, не мешая одному merge на закрытие milestone.

## Автоматизация

Переносимые **git-хуки** (`pre-commit`), переменная **`AEON_GIT_POLICY`**, связка с Cursor — см. **`GIT_HOOKS.md`**.
